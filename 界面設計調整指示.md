Vercel 會依照這些指令幫你 build。

## 第 2 步：確認 API URL 不要寫死 localhost

這一步很重要。

請改成環境變數，例如 Vite：

1. 登入 Render。
2. 點 **New +**。
3. 選 **Web Service**。
4. 連接 GitHub repository。
5. 選擇你的後端專案。

設定範例：

```
Build Command: npm install && npm run build
Start Command: npm run start
```

如果是 Express 且不需要 build：

```
Build Command: npm install
Start Command: npm start
```

在 Render 裡設定：

```env
JWT_SECRET=你的JWT密鑰
CORS_ORIGIN=https://你的前台.vercel.app
```

Render 設定環境變數後，可以觸發重新 build / deploy。[![](https://www.google.com/s2/favicons?domain=https://render.com&sz=128)Render](https://render.com/docs/configure-environment-variables?utm_source=chatgpt.com)

# 很重要：CORS 要打開前台網址

如果前台部署到 Vercel，後端部署到 Render，前台網址和後端網址不同，一定會遇到 CORS。

後端如果是 Express，可以設定：

```JavaScript
import cors from "cors";

app.use(cors({
  origin: [
    "http://localhost:5173",
    "https://你的前台.vercel.app"
  ],
  credentials: true
}));
```

不要為了方便直接長期使用：

```JavaScript
origin: "*"
```

因為如果你的系統有登入、cookie、token 或內部資料，這樣不安全。
