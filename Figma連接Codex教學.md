* **Output Directory**: `dist`

這些和 Vite 官方說的預設輸出位置一致。[![](https://www.google.com/s2/favicons?domain=https://vite.dev&sz=128)vitejs](https://vite.dev/guide/static-deploy)

### 如果有環境變數

如果你專案有 `.env` 變數，要在 Vercel 的 Project Settings → Environment Variables 補上。

### 點 Deploy

部署完成後你會拿到一個網址，直接給團隊測試。

### 之後怎麼更新

只要之後再：

```Bash
git add .
git commit -m "fix ui polish feedback"
git push
```

Vercel 就會自動重新部署。Vercel 也會替後續分支/提交產生預覽部署。[![](https://www.google.com/s2/favicons?domain=https://vite.dev&sz=128)vitejs](https://vite.dev/guide/static-deploy)

先用這個節奏最穩：

1. 先跑最後美化

## 兩個小提醒

第一，你現在這套如果還有上傳附件、下載、預覽這類行為，團隊測試前最好先再確認一次瀏覽器實測，因為 build 過不代表互動都穩。
